Project 2 — Key Wins

What Changed

Before (Legacy) → After (Lakehouse)
ODI + OJC + DataStage + PL/SQL → dbt SQL models (version-controlled)
Oracle DWH (licensed) → Spark + Delta Lake (open-source)
No version control → Full Git history + CI/CD
No automated testing → dbt tests + AI-powered verification
Zero documentation → Auto-generated data dictionary + DAG
Vendor lock-in ($$$) → 100% open-source stack

Cost Optimization

  • 🗑️DQ layer eliminated — saved ~340 table materializations (validation-only, no transformation)
  • 🔍Table pruning — 20–40% of tables typically found redundant during migration
  • 💻Local-first development — PostgreSQL dev target avoids cloud compute costs

Project 3 — Agent Network

9 Specialized AI Agents

Each agent is purpose-built for a specific role, collaborating through a shared Knowledge Base.

Lineage Extractor
Runs table lineage tracer, captures ODI/OJC metadata, outputs JSON
KB Loader
Loads migration data, classifies bronze sources (lookup vs. transactional)
Data Seeder
Seeds bronze tables from production with date-filtered extracts
Bronze Verifier
Validates row counts, sample records, numeric aggregates
Model Builder
Generates dbt SQL from legacy SQL — Oracle→PostgreSQL/Spark translation
Model Compiler
Runs dbt compile + dbt run, parses errors, resolves dependencies
Model Verifier
Cross-database comparison with numeric tolerance handling
Diagnosis Agent
Error pattern matching, SQL debugging, schema comparison
Plus: Pipeline Watchdog (higher-capability AI oversight)  +  System Services (OS-level process reliability)

Tiered AI Strategy

Cost-efficient models for deterministic DE tasks (SQL translation, ref injection, syntax conversion). Claude reserved for orchestration, diagnosis, and complex engineering decisions.
